## Z-Уровни

Z-уровни - это система, позволяющая объединять несколько карт в единую сеть связанных между собой вертикально уровней. Каждая карта в сети представляет собой "этаж", или определённую высоту. Между ними можно свободно перемещаться и просматривать верхние и нижние этажи, не перемещаясь на другую карту.
Примерные принципы работы:

### Сеть z-уровней
Для объединения всех карт, являющихся z уровнями, создаётся и используется пустая сущность с компонентом `CEZLevelsNetworkComponent`. Она хранит все уровни и их высоту.
Каждая карта в сети z-уровней имеет компонент `CEZLevelMapComponent`. Он хранит в себе высоту z-уровня.

### Многоэтажные гриды
Для создания многоэтажных гридов внутри сети используется компонент `GridMotionLinkComponent`. Он синхронизирует движения всех гридов и их позицию относительно центрального.
Внутри компонента имеется поле `GroupId`, по которому определяется причастность грида к своей группе.

### Наблюдатель
При присоединении игрока к сущности с компонентом `CEZLevelViewerComponent` на "ближайших" z-уровнях создаётся N число сущностей, чья позиция относительно карты синхронизируется с позицией основной сущности. Они используются для подгрузки сущностей с уровней, которые должны быть видны игроку.
Это позволяет рендерить все видимые уровни на клиенте внутри `ScalingViewport` корректно. Если этого компонента на сущности нет, игрок не будет видеть никакие z-уровни кроме текущего.

### Физика
За вертикальную физику отвечает компонент `CEZPhysicsComponent`. При инициализации сущности, в зависимости от того, соответствует ли она определённым требованиям, необходимым для корректной имитации физики, сущности добавляется или убирается компонент `CEActiveZPhysicsComponent`. Он является индикатором и не хранит внутри себя информацию.
`CEZPhysicsComponent` хранит в себе ряд переменных:
- `Velocity` представляет собой вертикальную скорость сущности.
- `LocalPosition` представляет собой вертикальную позицию сущности относительно карты. Значение данной переменной всегда находится между 0f и 1f включительно.
- `CurrentZLevel` представляет собой глобальную вертикальную позицию относительно сети z уровней. Она зависит от того, на какой карте находится сущность.
- `CurrentGroundHeight` представляет собой высоту земли на текущей позиции. Данная переменная используется для проверки расстояния между землёй и сущностью.
- `CurrentStickyGround` обозначает то, находится ли сущность в данный момент на лестнице.
- `Bounciness` представляет собой прыгучесть сущности.
- `GravityMultiplier` представляет собой модификатор гравитации, применяемой на сущность.
- `IsGrounded` обозначает то, находится ли сущность в данный момент на земле (или на лестнице).
- `IgnoreHighGround` обозначает то, игнорирует ли сущность лестницы и прочие подъёмы. Существует для наблюдателей.
- `NoRotDefault`, `DrawDepthDefault` и `SpriteOffsetDefault` являются клиентскими переменными, хранящими оригинальные значения для визуализации сущности.

Каждый кадр на клиенте и каждый тик на сервере происходит симуляция физики для z уровней. Симуляция происходит в несколько этапов:
1. Модификация вертикальной скорости
2. Движение
3. Обновление информации о том, находится ли сущность на земле
4. Перемещение между z-уровнями
5. Применение эффектов падения

**Примечание**
Вы можете узнать, находится ли сущность в данный момент на земле, используя `PhysicsComponent.IsGrounded()`, без необходимости проверять наличие `CEZPhysicsComponent`.

### Лестницы и возвышенности
Для функциональных возвышенностей используется компонент `CEZLevelHighGroundComponent`. Он не имеет функционала отдельно от перемещающейся сущности и нужен только для хранения определённых данных, а именно:
- `HeightCurve` - кривая, обозначающая дополнительную высоту при нахождении на объекте.
- `Stick` - является ли эта сущность "липучей". При значении true работает как лестница, при значении false - возвышенность, с которой можно упасть.
- `FixtureId` - fixture, используемый для определения нужного хитбокса.

Данный компонент используется при вычислении значения `CEZPhysicsComponent.CurrentGroundHeight`. В зависимости от позиции перемещающейся сущности получается определённое значение из кривой HeightCurve.

### Летающие сущности
Для того, чтобы сделать сущность летающей, к ней добавляется компонент `CEZFlyerComponent`. Он добавляет 3 действия - переключение режима полёта и выбор направления вверх/вниз. Сущность с этим компонентом всё ещё врезается в пол и потолок.
Для того, чтобы сущность могла свободно перемещаться, при этом игнорируя тайлы, используется компонент `CEZLevelGhostMoverComponent`. Он просто перемещает сущность по картам в сети z-уровней.

## Маппинг

Текущая система поддерживает маппинг практически идеально. Весь необходимый функционал присутствует в новых командах, единственная проблема - смещение гридов/карт.

### Создание сети
Для создания сети z-уровней используется команда `znetwork-combine <map1> <map2>...`. Для её использования необходимо указать все карты в порядке с самого нижнего по самый верхний этажи.
Данная команда вызывает метод, создающий новую сеть и добавляющий в неё все указанные карты в указанном порядке.

### Удаление карт
Для удаления всех карт и их сети z-уровней используется команда `znetwork-delete <network>`. Для её использования необходимо указать сущность сети. Все подходящие сущности будут находиться в подсказке в консоли.

### Инициализация карт
Для инициализации всех карт в сети z-уровней используется команда `znetwork-initialize <network>`. Для её использования необходимо указать сущность сети. Все подходящие сущности будут находиться в подсказке в консоли.
Данная команда не делает ничего кроме инициализации всех карт в сети.

### Сохранение карт
Для сохранения сети z-уровней используется команда `znetwork-savemap <network> <path>`. Для её использования необходимо указать сущность сети и путь, по которому будет сохранена карта.

**ОБРАТИТЕ ВНИМАНИЕ**
Сеть z-уровней сохраняется не в единый .yml файл, а в папку с указанным названием. Внутри данной папки будут сохранены все карты следующим образом:
- Каждой карте будет присвоено название `map-depth-{глубина}.yml`
- Будет создан файл `map-data.json`, внутри которого хранится информация о пути до каждого z-уровня

Следовательно, команда будет выглядеть примерно так:
`znetwork-savemap 32412 /Maps/_ECHO/MyMap`

### Загрузка карт
Для загрузки сети z-уровней используется команда `znetwork-loadmap <mapId> <path>`. Для её использования необходимо указать карту, на которую будет загружен нижний уровень и путь до **ПАПКИ** с сохранённой сетью.
Если mapId существующей пересечётся с mapId одного из загружаемых уровней, загрузка **БУДЕТ ПРЕРВАНА**.

### Погода
Для установки погоды на сеть z-уровней используется команда `znetwork-weather <network> <weather> <end time (необяз.)>`. Она работает так же, как weather, но использует сеть z-уровней.

### Передача энергии между этажами
Для передачи энергии между этажами используются прототипы по пути `Prototypes/_ECHO/Entities/Structures/Power/zcables.yml`.

### Небольшое замечание
Для правильной работы созданных вами карт, сделайте их планетами хотя бы на время маппинга. Так центр всех карт будет в одной и той же точке для всех уровней.

## Конфигурация через CCVars

```csharp
[CVarDefs]
public sealed partial class EchoCCVars
{
    /**
     * ZLevels
     */

    public static readonly CVarDef<float> ZImpactVelocityLimit =
        CVarDef.Create("zlevels.impact_velocity_limit", 0.75f, CVar.SERVER | CVar.REPLICATED | CVar.ARCHIVE);

    public static readonly CVarDef<int> MaxZLevelsBelowRendering =
        CVarDef.Create("zlevels.max_z_levels_below_rendering", 3, CVar.SERVER | CVar.REPLICATED | CVar.ARCHIVE);

    public static readonly CVarDef<float> ZLevelOffset =
        CVarDef.Create("zlevels.z_level_offset", .2f, CVar.SERVER | CVar.REPLICATED | CVar.ARCHIVE);
}
```


| CVar                                    |Значения|            Флаги            | Описание                                      |
|-----------------------------------------|--------|-----------------------------|-----------------------------------------------|
| `zlevels.impact_velocity_limit`         | `true` | SERVER, REPLICATED, ARCHIVE | Макс. скорость для урона от падения           |
| `zlevels.max_z_levels_below_rendering`  | `3`    | SERVER, REPLICATED, ARCHIVE | Число рендерящихся z-уровней                  |
| `barks.zlevels.z_level_offset`          | `0.2`  | SERVER, REPLICATED, ARCHIVE | Вертикальный сдвиг z-уровней                  |

**Флаги:**
- `SERVER` — значение хранится на сервере.
- `REPLICATED` — сервер отправляет значение клиентам (чтобы UI мог показать лимиты).
- `ARCHIVE` — значение сохраняется при перезапуске сервера.
- `CLIENTONLY` — настройка только клиента (каждый игрок настраивает сам).

## API

API (Application Programming Interface) в нашем случае — это набор методов, позволяющий сторонним системам взаимодействовать с системой z-уровней.
Система z-уровней предоставляет гибкий API, позволяющий свободно использовать систему как на стороне клиента, так и сервера.

### CESharedZLevelsSystem
`public bool TryGetZNetwork(EntityUid mapUid, [NotNullWhen(true)] out Entity<CEZLevelsNetworkComponent>? zLevel)`

Используется для получения сети z-уровней по сущности карты.


`public bool TryMapOffset(Entity<CEZLevelMapComponent?> inputMapUid, int offset, [NotNullWhen(true)] out Entity<CEZLevelMapComponent>? outputMapUid)`

Используется для получения z-уровня по определённому смещению.


`public bool TryMapUp(Entity<CEZLevelMapComponent?> inputMapUid, [NotNullWhen(true)] out Entity<CEZLevelMapComponent>? aboveMapUid)`

Используется для получения z-уровня сверху.


`public bool TryMapDown(Entity<CEZLevelMapComponent?> inputMapUid, [NotNullWhen(true)] out Entity<CEZLevelMapComponent>? belowMapUid)`

Используется для получения z-уровня снизу.


`public List<EntityUid> GetAllMapsAbove(Entity<CEZLevelMapComponent> inputMapUid)`

Используется для получения списка всех z-уровней сверху.


`public List<EntityUid> GetAllMapsBelow(Entity<CEZLevelMapComponent> inputMapUid)`

Используется для получения списка всех z-уровней снизу.


`public float DistanceToGround(Entity<CEZPhysicsComponent?> target)`

Используется для получения расстояния до земли для указанной сущности.


`public bool HasTileAbove(EntityUid ent, Entity<CEZLevelMapComponent?>? currentMapUid = null)`
`public bool HasTileAbove(Vector2i indices, Entity<CEZLevelMapComponent?> map)`

Используется для получения информации о том, есть ли над сущностью / на указанной позиции занятый тайл.


`public void SetZPosition(Entity<CEZPhysicsComponent?> ent, float newPosition)`

Используется для перемещения сущности по вертикали.


`public void SetZGravity(Entity<CEZPhysicsComponent?> ent, float newGravityMultiplier)`

Используется для назначения модификатора гравитации на сущность.
**Слишком большой модификатор раздавит сущность**


`public void SetZVelocity(Entity<CEZPhysicsComponent?> ent, float newVelocity)`

Используется для назначения вертикальной скорости для сущности.


`public void AddZVelocity(Entity<CEZPhysicsComponent?> ent, float newVelocity)`

Используется для добавления вертикальной скорости к текущей у сущности.


`public bool TryMove(EntityUid ent, int offset, Entity<CEZLevelMapComponent?>? map = null)`
`public bool TryMoveUp(EntityUid ent)`
`public bool TryMoveDown(EntityUid ent)`

Используется для перемещения сущности по z-уровням.


`public bool TryMoveDownOrChasm(EntityUid ent)`

Используется для перемещения сущности вниз. Если снизу нет карты, сущность упадёт в бездну.


`public bool HasOpaqueAbove(EntityUid ent, Entity<CEZLevelMapComponent?>? currentMapUid = null)`

Используется для получения информации о том, прозрачен ли тайл над сущностью.

### CEZLevelsSystem
`public Entity<CEZLevelsNetworkComponent> CreateZNetwork()`

Используется для создания сети z-уровней без карт внутри.


`public bool TryAddMapsIntoZNetwork(Entity<CEZLevelsNetworkComponent> network, Dictionary<EntityUid, int> maps)`

Используется для добавления карт в сеть z-уровней.


### CESharedZFlightSystem
`public bool TryActivateFlight(Entity<CEZFlyerComponent?> ent, CEZPhysicsComponent? zPhys = null)`

Используется для активации режима полёта у сущности.


`public void DeactivateFlight(Entity<CEZFlyerComponent?> ent, CEZPhysicsComponent? zPhys = null)`

Используется для деактивации режима полёта у сущности.


### Shared события
`CEZLevelMapMoveEvent`

Вызывается на сущность, когда она перемещается между z-уровнями.
Переменные:
- `int Offset` - то, сколько уровней было пройдено вместе с направлением.
- `int CurrentZLevel` - текущая высота.


`CEZLevelFallMapEvent`

Вызывается на сущность, когда она падает на уровень вниз.
Переменные отсутствуют.


`CEZLevelHitEvent`

Вызывается на сущность, когда она падает на землю.
Переменные:
- `float ImpactPower` - сила, с которой сущность упала на землю.


`CEGetZVelocityEvent`

Вызывается на сущность каждый кадр/тик для получения вертикальной скорости.
Переменные:
- `Entity<CEZPhysicsComponent> Target` - сущность и компонент физики.
- `float VelocityDelta` - скорость, которую можно модифицировать.


### Server события
`CEZLevelNetworkUpdatedEvent`

Вызывается на сеть z-уровней когда к ней добавляются/удаляются карты.


## Работа с прототипами

### Физика
```yml
  - type: CEZPhysics
    bounciness: 0.3             # Прыгучесть
    gravityMultiplier: 1        # Модификатор гравитации
    ignoreHighGround: false     # Игнорирует ли сущность лестницы
```

### Наблюдатель
```yml
  - type: CEZLevelViewer
    lookUp: true                # Может ли сущность смотреть вверх
```

### Лестницы
```yml
  - type: Physics               # Физика необходима для корректной работы
    bodyType: Static
  - type: Fixtures              # Хитбоксы, естественно, так же нужны
    fixtures:
      slowdown:
        shape:
          !type:PhysShapeAabb
          bounds: "-1.5,-0.4,1.5,0.4"
        layer:
          - SlipLayer
        mask:
          - ItemMask
        density: 1000
        hard: false
  - type: CEZLevelHighGround
    heightCurve:                # Кривая высоты. Располагайте сверху вниз, поворачивается вместе с объектами
    - 1.05
    - 1.05
    - 1.05
    - 0.5
    - 0.1
    stick: true                 # Лестница это, или другой объект. Если false, с объекта можно упасть
    fixtureId: slowdown         # Айди хитбокса, используемого для вычисления высоты
```

## FAQ

### Как лучше всего маппить с Z-уровнями?
Для маппинга с Z-уровнями лучше всего будет добавить на все карты в сети компонент `MapGridComponent`. Таким образом вы будете ставить тайлы одинаково ровно, как на планете.

### Как ставить лестницы?
Лестницы ставятся ТОЛЬКО на этаже снизу, обязательно с учётом визуально верного их поворота.

### Можно ли установить конкретный вес сущности?
Нет, можно только использовать модификатор гравитации и прыгучесть объекта.

### Нужно добавлять физику на все сущности?
Физику добавлять нужно только на не-статичные сущности. Если сущность не предполагает откручивание и какое-либо движение, вертикальная физика будет только нагружать сервер.

### Я хочу сделать объект, по которому можно ходить на этаже сверху
Для этого просто добавьте компонент `CEZLevelHighGroundComponent` и не редактируйте его свойства. Подобным образом уже можно ходить по стенам.
