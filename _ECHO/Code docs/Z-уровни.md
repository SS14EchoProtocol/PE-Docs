## Z-Уровни

Z-уровни - это система, позволяющая объединять несколько карт в единую сеть связанных между собой вертикально уровней. Каждая карта в сети представляет собой "этаж", или определённую высоту. Между ними можно свободно перемещаться и просматривать верхние и нижние этажи, не перемещаясь на другую карту.
Примерные принципы работы:

### Сеть z-уровней
Для объединения всех карт, являющихся z уровнями, создаётся и используется пустая сущность с компонентом CEZLevelsNetworkComponent. Она хранит все уровни и их высоту.
Каждая карта в сети z-уровней имеет компонент CEZLevelMapComponent. Он хранит в себе высоту z-уровня.

### Наблюдатель
При присоединении игрока к сущности с компонентом CEZLevelViewerComponent на "ближайших" z-уровнях создаётся N число сущностей, чья позиция относительно карты синхронизируется с позицией основной сущности. Они используются для подгрузки сущностей с уровней, которые должны быть видны игроку.
Это позволяет рендерить все видимые уровни на клиенте внутри ScalingViewport корректно. Если этого компонента на сущности нет, игрок не будет видеть никакие z-уровни кроме текущего. 

### Физика
За вертикальную физику отвечает компонент CEZPhysicsComponent. При инициализации сущности, в зависимости от того, соответствует ли она определённым требованиям, необходимым для корректной имитации физики, сущности добавляется или убирается компонент CEActiveZPhysicsComponent. Он является индикатором и не хранит внутри себя информацию.
CEZPhysicsComponent хранит в себе ряд переменных:
- Velocity представляет собой вертикальную скорость сущности.
- LocalPosition представляет собой вертикальную позицию сущности относительно карты. Значение данной переменной всегда находится между 0f и 1f включительно.
- CurrentZLevel представляет собой глобальную вертикальную позицию относительно сети z уровней. Она зависит от того, на какой карте находится сущность.
- CurrentGroundHeight представляет собой высоту земли на текущей позиции. Данная переменная используется для определения того, находится ли сущность на земле или в воздухе.
- CurrentStickyGround обозначает то, находится ли сущность в данный момент на лестнице.
- Bounciness представляет собой прыгучесть сущности.
- GravityMultiplier представляет собой модификатор гравитации, применяемой на сущность.
- IsGrounded обозначает то, находится ли сущность в данный момент на земле (или на лестнице).
- IgnoreHighGround обозначает то, игнорирует ли сущность лестницы и прочие подъёмы. Существует для наблюдателей.
- NoRotDefault, DrawDepthDefault и SpriteOffsetDefault являются клиентскими переменными, хранящими оригинальные значения для визуализации сущности.

Каждый кадр на клиенте и каждый тик на сервере происходит симуляция физики для z уровней. Симуляция происходит в несколько этапов:
1. Модификация вертикальной скорости
2. Движение
3. Обновление информации о том, находится ли сущность на земле
4. Перемещение между z-уровнями
5. Применение эффектов падения

### Лестницы и возвышенности
Для функциональных возвышенностей используется компонент CEZLevelHighGroundComponent. Он не имеет функционала отдельно от перемещающейся сущности и нужен только для хранения определённых данных, а именно:
- HeightCurve - кривая, обозначающая дополнительную высоту при нахождении на объекте.
- Stick - является ли эта сущность "липучей". При значении true работает как лестница, при значении false - возвышенность, с которой можно упасть.
- FixtureId - fixture, используемый для определения нужного хитбокса.

Данный компонент используется при вычислении значения CEZPhysicsComponent.CurrentGroundHeight. В зависимости от позиции перемещающейся сущности получается определённое значение из кривой HeightCurve.

## Маппинг

Текущая система практически полностью поддерживает маппинг. Весь необходимый функционал присутствует в новых командах.

### Создание сети
Для создания сети z-уровней используется команда `znetwork-combine <map1> <map2>...`. Для её использования необходимо указать все карты в порядке с самого нижнего по самый верхний этажи.
Данная команда вызывает метод, создающий новую сеть и добавляющий в неё все указанные карты в указанном порядке.

### Инициализация карт
Для инициализации всех карт в сети z-уровней используется команда `znetwork-initialize <network>`. Для её использования необходимо указать сущность сети. Все подходящие сущности будут находиться в подсказке в консоли.
Данная команда не делает ничего кроме инициализации всех карт в сети.

### Удаление карт
Для удаления всех карт и их сети z-уровней используется команда `znetwork-delete <network>`. Для её использования необходимо указать сущность сети. Все подходящие сущности будут находиться в подсказке в консоли.

### Сохранение карты
Для сохранения сети z-уровней используется команда `znetwork-savemap <network> <path>`. Для её использования необходимо указать сущность сети и путь, по которому будет сохранена карта.

**ОБРАТИТЕ ВНИМАНИЕ:** 
Сеть z-уровней сохраняется не в единый .yml файл, а в папку с указанным названием. Внутри данной папки будут сохранены все карты следующим образом: 
- Каждой карте будет присвоено название `map-depth-{глубина}.yml`
- Будет создан файл `map-data.json`, внутри которого хранится информация о пути до каждого z-уровня

Если вы укажете папку, внутри которой уже есть файлы, сеть **НЕ** будет сохранена.

### Загрузка карты
Для загрузки сети z-уровней используется команда `znetwork-loadmap <mapId> <path>`. Для её использования необходимо указать карту, на которую будет загружен нижний уровень и путь до **ПАПКИ** с сохранённой сетью. 
Если mapId существующей пересечётся с mapId одного из загружаемых уровней, загрузка **БУДЕТ ПРЕРВАНА**.

## Конфигурация через CCVars

```csharp
[CVarDefs]
public sealed partial class EchoCCVars
{
    /**
     * ZLevels
     */

    public static readonly CVarDef<float> ZImpactVelocityLimit =
        CVarDef.Create("zlevels.impact_velocity_limit", 0.75f, CVar.SERVER | CVar.REPLICATED | CVar.ARCHIVE);

    public static readonly CVarDef<int> MaxZLevelsBelowRendering =
        CVarDef.Create("zlevels.max_z_levels_below_rendering", 3, CVar.SERVER | CVar.REPLICATED | CVar.ARCHIVE);

    public static readonly CVarDef<float> ZLevelOffset =
        CVarDef.Create("zlevels.z_level_offset", .2f, CVar.SERVER | CVar.REPLICATED | CVar.ARCHIVE);
}
```

**Что это:** Конфигурационные переменные (CVars) — настройки, которые админ может менять без рекомпиляции сервера.

**Каждая переменная:**

| CVar                                    |Значения|            Флаги            | Описание                                      |
|-----------------------------------------|--------|-----------------------------|-----------------------------------------------|
| `zlevels.impact_velocity_limit`         | `true` | SERVER, REPLICATED, ARCHIVE | Макс. скорость для урона от падения           |
| `zlevels.max_z_levels_below_rendering`  | `3`    | SERVER, REPLICATED, ARCHIVE | Число рендерящихся z-уровней                  |
| `barks.zlevels.z_level_offset`          | `0.2`  | SERVER, REPLICATED, ARCHIVE | Вертикальный сдвиг z-уровней                  |

**Флаги:**
- `SERVER` — значение хранится на сервере.
- `REPLICATED` — сервер отправляет значение клиентам (чтобы UI мог показать лимиты).
- `ARCHIVE` — значение сохраняется при перезапуске сервера.
- `CLIENTONLY` — настройка только клиента (каждый игрок настраивает сам).
